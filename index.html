Software Engineering & Design Artifact

Travlr Getaways (MEAN stack) enhanced to meet the CS-499 software engineering and design outcomes.

1. Backend Architecture & API Design

Category One

The Node.js/Express API is organized into well-defined layers: routes, controllers, services, and data access. Each layer has a single responsibility, making the code easier to maintain, test, and extend.

I use RESTful conventions for endpoints, clear request/response models, and centralized error handling to keep the API predictable and consistent.

  • Express routing
  • Controller–service pattern
  • RESTful endpoints
  • Error handling

2. Domain Modeling & Data Design

Software Design

Trip, user, and booking data are modeled with Mongoose schemas that match the real-world domain: trip codes, pricing, dates, and relationships between travelers and trips.

Validation rules and schema constraints help enforce data quality at the model layer, reducing bugs and keeping the database consistent.

  • MongoDB / Mongoose
  • Domain modeling
  • Validation rules
  • Schema design

3. Backend Security & Reliability Engineering

Security & SE Design

This enhancement improves the security and reliability of the Travlr API. I implemented consistent middleware-based authentication checks, safe password handling, standardized error responses, and server-side input validation.

These improvements strengthen system integrity and demonstrate professional software engineering practices by ensuring predictable behavior, reducing risk of bad data, and preventing unauthorized access.

  • JWT authentication
  • Input validation
  • Error handling
  • Middleware design
