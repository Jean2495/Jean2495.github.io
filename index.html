Software Engineering & Design Artifact

Travlr Getaways (MEAN stack) enhanced to meet the CS-499 software engineering and design outcomes.

1. Backend Architecture & API Design

Category One

The Node.js/Express API is organized into well-defined layers: routes, controllers, services, and data access. Each layer has a single responsibility, making the code easier to maintain, test, and extend.

I use RESTful conventions for endpoints, clear request/response models, and centralized error handling to keep the API predictable and consistent.

  • Express routing
  • Controller–service pattern
  • RESTful endpoints
  • Error handling

2. Domain Modeling & Data Design

Software Design

Trip, user, and booking data are modeled with Mongoose schemas that match the real-world domain: trip codes, pricing, dates, and relationships between travelers and trips.

Validation rules and schema constraints help enforce data quality at the model layer, reducing bugs and keeping the database consistent.

  • MongoDB / Mongoose
  • Domain modeling
  • Validation rules
  • Schema design

3. Backend Security & Reliability Engineering

Security & SE Design

This enhancement improves the security and reliability of the Travlr API. I implemented consistent middleware-based authentication checks, safe password handling, standardized error responses, and server-side input validation.

These improvements strengthen system integrity and demonstrate professional software engineering practices by ensuring predictable behavior, reducing risk of bad data, and preventing unauthorized access.

  • JWT authentication
  • Input validation
  • Error handling
  • Middleware design

Algorithms & Data Structures Enhancement

Category Two enhancement demonstrating optimized data retrieval using cursor-based pagination, indexed filtering, and efficient query algorithms.

1. Cursor-Based Pagination

Category Two

I replaced the original skip()-based pagination with a cursor-driven algorithm using MongoDB’s ObjectId values. This avoids scanning and skipping large sets of documents and greatly improves time complexity on growing datasets.

The backend now returns nextCursor tokens, enabling stable, predictable, and efficient forward pagination.

  • Improved time complexity
  • Efficient iteration
  • Cursor decoding & encoding
  • Scalable API design

2. Indexed Filtering & Query Optimization

Performance

The search API now supports efficient filtering by resort, price, and date ranges. These filters leverage model-level indexes to reduce lookup times and avoid full collection scans.

I implemented dynamic filter construction based on the user’s search parameters to minimize query cost.

  • Compound indexes
  • Date-range search
  • Efficient filters
  • Reduced query overhead

3. Algorithmic Design & Documentation

DSA

I designed a clean search algorithm that validates inputs, builds optimized queries, and handles cursor markers for pagination. The final version is documented with pseudocode to explain logic flow.

This demonstrates professional algorithmic reasoning aligned with CS-499 requirements.

  • Pseudocode design
  • Query planning
  • Edge-case handling
  • Validation logic

Enhancement Timeline

How the Travlr Getaways project has evolved into a fully enhanced, capstone-ready artifact across software engineering, algorithms, and databases.
